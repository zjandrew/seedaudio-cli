# 音色 · 语气 · 文本规范化参考

写合成文本、选音色、给语气时查这里。**音色 id 以[控制台音色库](https://console.volcengine.com/speech/new/voices)为准**——本文列的是常见示例,可能过时;用前不确定就用 `seedaudio-cli voices` 看精选子集,或让用户去控制台确认。

---

## 1. 音色场景对照(示例)

| 场景 | 想要的气质 | 候选方向 | 配套参数 |
|---|---|---|---|
| 有声书 / 长篇旁白 | 沉稳、耐听、不抢戏 | 中性男声 / 温润女声(`*_uranus_bigtts` / `*_mars_bigtts`) | `-m standard` 求稳,段间 `--silence-ms 300` |
| 新闻 / 播报 | 专业、清晰、中正 | 播报男声 / 主播女声 | `--instruct "专业清晰的播报语气"`,`--speech-rate 5~15` |
| 客服 / 提示音 | 明亮、亲和 | 客服女声(如"灿灿"系) | 短句,`--speech-rate 0`,音量适中 |
| 广告 / 营销 | 有感染力、情绪足 | 表现力强的音色 | `-m expressive` + 情绪 `--instruct`,关键词前后留停顿 |
| 儿童 / 活泼 | 俏皮、上扬 | 活泼女声 | `-m expressive`,`--instruct "活泼俏皮、上扬"`,`--pitch 1~3` |
| 悬疑 / 旁白叙事 | 低沉、神秘 | 低沉男声 | `--instruct "压低声音、带神秘感"`,`--speech-rate -10` |
| 角色对话 | 每角色一个固定音色 | 男/女/老/少分开选 | 统一编码/采样率,逐句分别合成再拼 |

**选音色三原则**:
1. 一个"角色/栏目"全程**固定一个音色**,换音色 = 换人。
2. 不确定 id 就 `voices` 查或让用户去控制台,别瞎填。
3. 复刻音色(`S_...`)走 `--resource-id seed-icl-2.0`,且用指令/标签时加 `-m seed-tts-2.0-expressive`。

## 2. 语气指令库(`--instruct`)

模板:`用<情绪><强度>的语气,<语速/音量/节奏提示>说`。一条聚焦一种语气。

| 情绪 | 指令示例 |
|---|---|
| 平述/中性 | `用平和、自然、中正的语气说` |
| 开心 | `用特别开心、上扬雀跃的语气说` |
| 撒娇 | `用撒娇、软糯、略带鼻音的语气说` |
| 温柔安抚 | `用温柔、放慢、带安抚感的语气说` |
| 痛心/难过 | `用低沉、缓慢、带哽咽感的痛心语气说` |
| 愤怒 | `用压抑又强硬、咬字加重的语气说` |
| 兴奋激动 | `用高昂、语速偏快、富有感染力的语气说` |
| 专业播报 | `用专业、清晰、中正的新闻播报语气说` |
| 神秘悬念 | `用压低声音、放慢、带神秘感的语气说` |
| 温暖治愈 | `用温暖、舒缓、像睡前故事一样的语气说` |

要点:
- 官方 2.0 音色可直接用;复刻音色需 `-m seed-tts-2.0-expressive`。
- 强情感波动大时,降低预期或回退 `standard`;同一段别频繁切语气。
- 情绪要"具象"而非抽象:与其"很伤心",不如"低沉、缓慢、句尾下沉、带停顿"。

## 3. 文本规范化清单(读对的前提)

把"写法"改成"读法",交付时在"优化问题"里披露改了什么。

### 数字
| 写法 | 读法 | 说明 |
|---|---|---|
| `3.14` | 三点一四 | 小数逐位 |
| `2026 年` | 二零二六年 | 年份逐位 |
| `2026 元` | 两千零二十六元 | 数量按数值 |
| `第 3 章` | 第三章 | 序数 |
| `13800138000` | 幺三八零零幺三八零零零 | 电话逐位,1 常读"幺" |
| `1/2` | 二分之一 | 分数 |
| `-5°C` | 零下五摄氏度 | |

### 金额 / 单位 / 时间
`$5`→五美元;`￥99`→九十九元;`5kg`→五千克;`5km`→五公里;`5%`→百分之五;`5‰`→千分之五;`12:30`→十二点三十分;`08:00`→上午八点。

### 符号
`&`→和;`@`→艾特(或"在");`#`→井号(或"第");`/`→每/斜杠(按语义);`~`→到(区间)或波浪号;`—`→停顿延展;`...`→省略号拖顿;`()`→括号内可视为补充,默认朗读,要过滤用 `additions.max_length_to_filter_parenthesis`。

### 缩写 / 外文
`API`→保留或"诶屁挨";`CEO`→保留或"首席执行官";`Mr.`→先生;`vs`→对;`OK`→欧凯/好的。**中英混读**用 `additions.explicit_language=zh-cn`。

### 多音字 / 形近字兜底
模型偶尔读错多音字(行/重/长/参/差…)、生僻字、形近字。处理:
1. 上下文足够时多数能读对,不必动。
2. 确实读错的:换发音一致的常用同音字(如某生僻地名→同音常用字),并披露替换。
3. 关键专名(人名/品牌)读错风险高时,提醒用户核对或提供拼音备注。

### Markdown / Emoji
默认过滤 Markdown 语法与 emoji。要原样朗读星号等:`--from-json` 传 `additions.disable_markdown_filter=false`;要保留 emoji 朗读:`additions.disable_emoji_filter=false`。

## 4. 停顿与节奏工具

| 手段 | 用法 | 适合 |
|---|---|---|
| 标点 | 逗号短停、句号长停、省略号拖顿、破折号延展 | **首选,最稳** |
| 句尾留白 | `--silence-ms 200~500` | 段落衔接、呼吸 |
| 段间静音 | 分段时每段加 `--silence-ms`,或拼接时插静音文件 | 长文分段 |
| SSML `<break/>` | `--ssml` + `<break time="300ms"/>` | 中英文音色、需精确停顿 |
| 语速 | `--speech-rate`(整体) | 全段偏快/偏慢 |

> 别用"空格""换行"指望停顿——靠标点和 `--silence-ms`/SSML。

## 5. 进阶参数(走 `--from-json`)

v1 没做成 flag 的字段,放进一个 JSON 文件当 `req_params` 基底,flag 仍可覆盖顶层:

```json
{
  "additions": {
    "explicit_language": "zh-cn",
    "explicit_dialect": "",
    "silence_duration": 300,
    "disable_markdown_filter": true,
    "max_length_to_filter_parenthesis": 0
  },
  "use_tag_parser": false
}
```

```bash
seedaudio-cli synthesize --from-json base.json -p "文本" --voice <id> --out a.mp3
```

> `additions` 的精确嵌套以官方文档为准,先 `--dry-run` 打印请求体核对,再用真实凭证联调。`use_tag_parser`(cot 标签,如 `<cot text=急促难耐>…</cot>`)只对复刻音色 + `-m seed-tts-2.0-expressive` 生效,且标签内文本宜短(<64 字符)。
